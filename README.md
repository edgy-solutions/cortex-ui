## Overview

Cortex UI serves as the primary human-in-the-loop interface for the **Invincible Agent** polyglot microservice architecture. Rather than relying on static dashboards or standard chatbot text, Cortex UI implements an **Intent-Based Semantic UI**. 

It converses with the orchestration gateway, streams execution traces from the agent mesh, and dynamically renders React components based on strict BAML data contracts generated by the Presentation Agent (Engine F).

## Architectural Highlights

### 1. Server-Driven UI (Semantic Translation)
The UI is completely decoupled from the underlying data sources. It relies on the **Presentation Agent (Engine F)** to translate raw JSON payloads from upstream expert agents into abstract `DashboardUI` archetypes. The frontend simply maps these archetypes to specific React components:

| Archetype | Description | React Component | Layout |
|-----------|-------------|-----------------|--------|
| `PROCESS_TOPOLOGY` | Workflow graphs with nodes and edges. Triggers the interactive blueprint phase. | React Flow (`@xyflow/react`) | Full-width |
| `HAZARD_DECLARATION` | Risk alerts, compliance violations, and safety warnings with severity badges. | WarningCard | Inline (2-col grid) |
| `ASSET_STATE_METRIC` | Entity/Type/Detail tables for telemetry, inventory, metadata lists, or tool lists. | SupplyTable | Inline (2-col grid) |
| `KNOWLEDGE_DOCUMENT` | Technical procedure overviews, manual drafts, and audit reports. | Markdown (`react-markdown`) | Full-width |
| `CHART_WIDGET` | Interactive data visualizations (e.g., Superset query results) with dynamic chart types. | Recharts Widget | Full-width / Inline |

### 2. Resilient SSE Stream Handling
Cortex UI communicates with the Orchestration Gateway (Engine G) via Server-Sent Events (SSE). To handle long-running agentic workflows (which can take minutes to resolve) and prevent "Retry Storms", the frontend implements a robust connection manager using `@microsoft/fetch-event-source`:
- **Custom `onopen` Handlers:** Intercepts HTTP errors (like 401 Unauthorized) before stream initialization to fail gracefully and redirect to authentication.
- **`AbortController` Integration:** Explicitly aborts the fetch request upon receiving a `stream_end` event, preventing the library's default behavior of infinitely reconnecting and spawning duplicate Dagster runs.
- **React Query Kill-Switch:** Integrates with `@tanstack/react-query` using `retry: false` to ensure asynchronous chaos is mitigated.

### 3. Runtime Configuration Injection
To ensure a single, immutable OCI image can be promoted across environments (Dev, Staging, Prod) without rebuilding, Cortex UI uses a **Runtime Config Injection** pattern:
1. Vite bakes `import.meta.env` variables into the static JS bundle at build time.
2. In Kubernetes, an `initContainer` reads the live environment variables and generates a `/workspace/dist/config.js` file.
3. `index.html` loads this script before the main application bundle.
4. `src/config.ts` acts as a central router, prioritizing the injected runtime config over the build-time static config.

## Quick Start

### Prerequisites
- Node.js 20+
- npm

### Installation

```bash
git clone <repo-url>
cd cortex-ui
npm install
```

### Environment Configuration

Create a `.env` file at the project root:

```env
# API Gateway URL
VITE_API_URL=http://localhost:8888

# Keycloak OIDC Configuration
VITE_KEYCLOAK_REALM_URL=http://localhost:8080/realms/cortex
VITE_KEYCLOAK_CLIENT_ID=cortex-ui

# Set to true to bypass authentication for local development
VITE_NO_AUTH=false
```

### Running Locally

```bash
npm run dev
```

Open [http://localhost:5173](http://localhost:5173) in your browser.

## Authentication

Cortex UI uses **OpenID Connect (OIDC)** via Keycloak for secure, enterprise-grade access control.
- **`RequireAuth`**: A higher-order component that wraps the application, ensuring only authenticated users can access the mesh.
- **Bearer Tokens**: JWT tokens are automatically attached to all REST and SSE requests.
- **Session Management**: Expired tokens (401s) are gracefully caught by the SSE handler, prompting a clean redirect to the login portal.

## Project Structure

```
src/
├── api/                    # API client, SSE stream parser, BAML types
├── store/                  # Zustand state management (messages, phase, ontology)
├── hooks/                  # React Query mutations and agent interaction logic
├── components/
│   ├── NeuralStream/       # Chat UI, thinking cards, and agent assembly loaders
│   ├── HUD/                # Sidebar: real-time ontology map and data bindings
│   ├── Blueprint/          # React Flow components for PROCESS_TOPOLOGY
│   ├── mesh/               # Semantic UI widgets (ChartWidget, SupplyTable, etc.)
│   └── registry/           # SemanticInterpreter mapping archetypes to components
```
